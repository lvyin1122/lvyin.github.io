Concurrency Control

2020-02-10 Notes Operating Systems Comments

Concurrency Control

Intro

processes:

  • share data
  • executed in certain order

this will cause concurrency control problems

Shared Data

Synchronization of Concurrent processes

Problems:

  • how to ensure shared data correctly read/written
  • how to synchronize these processes

Solutions:
concurrency control methods

Atomic Instruction

Two properties:

  1. executed to completion without interruption
  2. if two processes execute the same, the execution is sequential

Critical Section Problem

each process has a code segment, called critical section, which accesses the shared data

Solution 1: Test-and-set

  • Wait until the lock of the critical section is not locked, and them lock the lock
  • Execute the critical section
  • Unlock the lock

TestAndSet is a headware atomic instruction to manage the lock

if the lock is locked
then return  FALSE and lock the lock
else return TRUE

For a process:

do{
    while(TesAndSet(lock));
    critical section;
    lock = FALSE;
    remainder;
}while(TRUE);

Solution 2: Semaphore

Let S be the number of available keys for the toilet

do{
    A user waits until getting the key;
    while(S<=0);
    S=S-1;
    Enter and use the toilet;
    The user returns the key and signals the other users;
    S=S+1;
    Do other things;
}while (TRUE)

Process synchronization

P1: SolveEquation1(); then signal(S);
P2: waits(S); then SolveEquation2();

Resource Sharing

How can N processes share M units of resources
Initialized S to M
For a process:

waits(S);
Acquire one unit of resource and use it until completion;
signal(S);

Producer-Consumer Problem

  • A producer produces data and writes it to a buffer
  • A concumer reads & removes this data from the shared buffer and concumes it

Problem: big data with small shared buffer
Solution:

  • Mutual exclusion
  • Writing
  • Reading

Using Semaphore:

  • mutex: binary semaphore
  • empty: integer semaphore
  • occupied: counts the number of occupied buffer spaces

Producer-Main Idea:

Producer:

do{
    Produce a data item;
    wait(empty);
    wait(mutex);
    Write the data item to the buffer;
    signal(mutex);
    signal(occupied);
}while(TRUE)

Consumer:

do{
    wait(occupied);
    wait(mutex);
    Read and remove a data item from buffer;
    signal(mutex);
    signal(empty);
    Consume the data item;
}while(TRUE)
